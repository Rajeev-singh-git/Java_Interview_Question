# 07.1 Introduction to Streams

The **Stream API** is one of the most important features introduced in **Java 8**. It provides a modern way to process collections using **Functional Programming** concepts such as **Lambda Expressions** and **Method References**.

Instead of writing explicit loops, Stream API allows us to express **what** we want to achieve rather than **how** to achieve it.

---

# Why were Streams Introduced?

Before Java 8, collections were typically processed using loops.

Suppose we want to print all even numbers.

```java
List<Integer> numbers = List.of(2, 5, 8, 11, 14);

for (Integer number : numbers) {
    if (number % 2 == 0) {
        System.out.println(number);
    }
}
```

As business requirements grow, the code becomes more difficult to read.

For example:

* Filter employees
* Sort them by salary
* Convert names to uppercase
* Collect the results

Using nested loops and temporary collections quickly becomes verbose.

Java 8 introduced the **Stream API** to make collection processing more expressive and readable.

The same example using Streams:

```java
numbers.stream()
       .filter(n -> n % 2 == 0)
       .forEach(System.out::println);
```

Notice that the code focuses on **what** should happen rather than **how** to iterate through the collection.

---

# What is a Stream?

A **Stream** is a sequence of elements that supports operations for processing data.

Unlike a collection, a Stream **does not store data**.

Instead, it processes data from a source such as:

* Collection
* Array
* File
* Generator
* Infinite sequence

Think of a Stream as a **pipeline** through which data flows and gets processed.

```text
Collection
      │
      ▼
    Stream
      │
Processing Operations
      │
      ▼
    Result
```

---

# Stream vs Collection

Although Streams are usually created from Collections, they are fundamentally different.

| Collection                      | Stream                    |
| ------------------------------- | ------------------------- |
| Stores data                     | Processes data            |
| Can add or remove elements      | Cannot modify the source  |
| Can be traversed multiple times | Can be consumed only once |
| Uses external iteration         | Uses internal iteration   |

A Collection is a **data structure**.

A Stream is a **data processing pipeline**.

---

# External Iteration vs Internal Iteration

This is one of the biggest conceptual changes introduced by the Stream API.

## External Iteration

Before Java 8, developers controlled the iteration manually.

```java
for (Integer number : numbers) {
    System.out.println(number);
}
```

Here, **you** decide:

* when to start
* when to stop
* how to iterate

This is called **External Iteration**.

---

## Internal Iteration

With Streams, iteration is handled internally.

```java
numbers.stream()
       .forEach(System.out::println);
```

You simply describe the operation.

The Stream API decides **how** to iterate over the data.

This is called **Internal Iteration**.

One advantage of internal iteration is that the Stream API can optimize execution, including parallel processing.

---

# Stream Pipeline

Every Stream follows the same processing pipeline.

```text
Source
(Collection, Array, File...)
          │
          ▼
Create Stream
          │
          ▼
Intermediate Operations
(filter, map, sorted...)
          │
          ▼
Terminal Operation
(collect, count, reduce...)
          │
          ▼
Result
```

Example:

```java
List<String> names =
        List.of("java", "spring", "docker");

List<String> result =
        names.stream()                     // Create Stream
             .map(String::toUpperCase)     // Intermediate Operation
             .sorted()                     // Intermediate Operation
             .toList();                    // Terminal Operation
```

A Stream pipeline consists of three stages:

1. **Source** – Where the data comes from.
2. **Intermediate Operations** – Transform the data.
3. **Terminal Operation** – Produce the final result.

Without a **Terminal Operation**, the pipeline is never executed.

---

# Why Streams are Powerful

The Stream API provides several advantages over traditional loops.

* Less boilerplate code.
* Better readability.
* Declarative programming style.
* Easy chaining of multiple operations.
* Works seamlessly with Lambda Expressions and Method References.
* Supports parallel processing through Parallel Streams.

Streams encourage developers to focus on **business logic** rather than iteration logic.

---

# Creating Streams

Streams can be created from various data sources.

## From a Collection

```java
List<String> names = List.of("Java", "Spring", "Docker");

Stream<String> stream = names.stream();
```

---

## From an Array

```java
String[] languages = {"Java", "Python", "Go"};

Stream<String> stream =
        Arrays.stream(languages);
```

---

## Using `Stream.of()`

```java
Stream<String> stream =
        Stream.of("Java", "Spring", "Docker");
```

---

## Infinite Streams

```java
Stream.generate(Math::random)
      .limit(5)
      .forEach(System.out::println);
```

---

# Characteristics of Streams

## 1. Streams do not store data.

They process data from a source such as a Collection or an Array.

---

## 2. Streams do not modify the source.

```java
List<String> names =
        List.of("java", "spring");

names.stream()
     .map(String::toUpperCase)
     .toList();
```

The original list remains unchanged.

---

## 3. Streams can be consumed only once.

```java
Stream<Integer> stream =
        List.of(1, 2, 3).stream();

stream.count();

stream.forEach(System.out::println);   // ❌ IllegalStateException
```

After a terminal operation, the Stream is closed.

---

## 4. Streams support functional programming.

Operations are typically implemented using:

* Lambda Expressions
* Method References
* Functional Interfaces

---

# Lazy Evaluation

Intermediate operations are **lazy**.

They are not executed immediately.

```java
List<Integer> numbers =
        List.of(2, 5, 8, 11);

numbers.stream()
       .filter(n -> {
           System.out.println("Filtering " + n);
           return n % 2 == 0;
       });
```

Nothing is printed because there is no terminal operation.

Now add one:

```java
numbers.stream()
       .filter(n -> n % 2 == 0)
       .count();
```

The pipeline executes only when the terminal operation is reached.

This behavior improves performance by avoiding unnecessary work.

# Real-world Usage

Stream API is commonly used for:

* Filtering collections
* Data transformation
* Sorting
* Removing duplicates
* Aggregation
* Reporting
* Processing database results
* Processing API responses

---

# Best Practices

* Prefer Streams for collection processing.
* Keep stream pipelines short and readable.
* Use Method References when they improve readability.
* Avoid modifying external state inside stream operations.
* Choose loops instead of Streams when they make the code simpler.

---

# Common Mistakes

* Reusing an already consumed Stream.
* Forgetting the terminal operation.
* Writing long and difficult-to-read stream pipelines.
* Using Streams for every problem, even when a simple loop is clearer.

---

# Interview Questions

### 1. Why was the Stream API introduced?

To process collections in a declarative, functional, and more readable manner while reducing boilerplate code.

---

### 2. What is the difference between a Stream and a Collection?

A Collection stores data.

A Stream processes data.

---

### 3. What is the difference between External Iteration and Internal Iteration?

With External Iteration, the developer controls iteration using loops.

With Internal Iteration, the Stream API manages the iteration.

---

### 4. Can a Stream be reused?

No.

A Stream can be consumed only once.

---

### 5. What is lazy evaluation?

Intermediate operations are executed only when a terminal operation is invoked.

---

### 6. Does a Stream modify the original Collection?

No.

A Stream processes data without modifying the source collection.
