# 07.3 Collectors

The `collect()` operation is the most commonly used **Terminal Operation** in the Stream API.

Its purpose is to **accumulate** the elements of a Stream into another form such as a `List`, `Set`, `Map`, or a summarized result.

The actual accumulation logic is provided by the **Collectors** utility class.

---

# Why do we need Collectors?

Suppose we want to convert all names to uppercase and store them in a `List`.

```java
List<String> names =
        List.of("java", "spring", "docker");
```

Without Collectors, there is no straightforward way to accumulate the processed elements.

Using `collect()`:

```java
List<String> result =
        names.stream()
             .map(String::toUpperCase)
             .collect(Collectors.toList());
```

The Stream processes the data.

`Collectors` decide **how the final result should be accumulated**.

---

# How collect() Works

```text
Stream
    │
Processes Data
    │
    ▼
collect()
    │
Uses
Collectors
    │
    ▼
Final Result
(List, Set, Map...)
```

Think of it this way:

* Stream → Processes data
* Collectors → Decide how to store the processed data

---

# collect() vs Collectors

This is a common source of confusion.

| `collect()`                  | `Collectors`                         |
| ---------------------------- | ------------------------------------ |
| Terminal Operation           | Utility class                        |
| Executes the Stream pipeline | Provides accumulation strategies     |
| Called on a Stream           | Passed as an argument to `collect()` |

Example:

```java
names.stream()
     .collect(Collectors.toList());
```

Here,

* `collect()` is the Stream method.
* `Collectors.toList()` tells `collect()` to return a `List`.

---

# Collecting into a List

The most common collector is `toList()`.

```java
List<String> result =
        names.stream()
             .map(String::toUpperCase)
             .collect(Collectors.toList());
```

Result

```text
[JAVA, SPRING, DOCKER]
```

> **Note:** From Java 16 onwards, `toList()` can also be called directly on the Stream.

```java
List<String> result =
        names.stream()
             .map(String::toUpperCase)
             .toList();
```

---

# Collecting into a Set

Use `toSet()` when duplicate elements should be removed.

```java
List<String> names =
        List.of("Java", "Spring", "Java");

Set<String> result =
        names.stream()
             .collect(Collectors.toSet());
```

Result

```text
[Java, Spring]
```

---

# Collecting into a Map

`toMap()` converts Stream elements into key-value pairs.

```java
List<String> names =
        List.of("Java", "Spring", "Docker");

Map<String, Integer> result =
        names.stream()
             .collect(Collectors.toMap(
                     name -> name,
                     String::length
             ));
```

Result

```text
{
 Java=4,
 Spring=6,
 Docker=6
}
```

Common use cases:

* Employee ID → Employee
* Product ID → Product
* Username → User

> **Note:** Duplicate keys cause an `IllegalStateException` unless a merge function is provided.

---

# joining()

`joining()` combines multiple Strings into a single String.

```java
String result =
        names.stream()
             .collect(Collectors.joining(", "));
```

Output

```text
Java, Spring, Docker
```

You can also specify a prefix and suffix.

```java
String result =
        names.stream()
             .collect(
                     Collectors.joining(
                             ", ",
                             "[",
                             "]"
                     )
             );
```

Output

```text
[Java, Spring, Docker]
```

Common use cases:

* CSV generation
* Reports
* Displaying comma-separated values

---

# counting()

`counting()` counts the number of elements in a Stream.

```java
Long total =
        names.stream()
             .collect(Collectors.counting());
```

Output

```text
3
```

This is similar to:

```java
long total =
        names.stream()
             .count();
```

However, `Collectors.counting()` is especially useful inside collectors like `groupingBy()`, which you'll see in the next section.

---

# Collector Summary

| Collector    | Purpose         |
| ------------ | --------------- |
| `toList()`   | Convert to List |
| `toSet()`    | Convert to Set  |
| `toMap()`    | Convert to Map  |
| `joining()`  | Combine Strings |
| `counting()` | Count elements  |

---

# groupingBy()

`groupingBy()` groups Stream elements based on a given classification function.

Suppose we have:

```java
class Employee {

    private String name;
    private String department;

}
```

```java
List<Employee> employees = ...
```

Group employees by department.

```java
Map<String, List<Employee>> result =
        employees.stream()
                 .collect(
                     Collectors.groupingBy(
                         Employee::getDepartment
                     )
                 );
```

Result

```text
IT      → [Employee1, Employee2]
HR      → [Employee3]
Finance → [Employee4, Employee5]
```

Common use cases:

- Employees by department

- Students by grade

- Products by category

- Orders by customer

---

# partitioningBy()

`partitioningBy()` divides elements into **exactly two groups** based on a condition.

It always returns a `Map<Boolean, List<T>>`.

Example:

```java
Map<Boolean, List<Employee>> result =
        employees.stream()
                 .collect(
                     Collectors.partitioningBy(
                         employee ->
                             employee.getSalary() > 50000
                     )
                 );
```

Result

```text
true  → Employees with salary > 50000

false → Remaining employees
```

Use `partitioningBy()` when there are only **two possible groups**.

---

# groupingBy() vs partitioningBy()

| groupingBy()              | partitioningBy()                 |
| ------------------------- | -------------------------------- |
| Creates multiple groups   | Creates only two groups          |
| Returns `Map<K, List<T>>` | Returns `Map<Boolean, List<T>>`  |
| Groups using a key        | Groups using a boolean condition |

---

# mapping()

`mapping()` transforms elements **inside another collector**.

Example:

```java
Map<String, List<String>> result =
        employees.stream()
                 .collect(
                     Collectors.groupingBy(
                         Employee::getDepartment,
                         Collectors.mapping(
                             Employee::getName,
                             Collectors.toList()
                         )
                     )
                 );
```

Instead of collecting entire `Employee` objects, only employee names are collected.

Result

```text
IT      → [Raj, Amit]

HR      → [Neha]
```

---

# summingInt()

Calculates the sum of integer values.

```java
int totalSalary =
        employees.stream()
                 .collect(
                     Collectors.summingInt(
                         Employee::getSalary
                     )
                 );
```

Common use cases:

- Total salary

- Total marks

- Total quantity

---

# averagingInt()

Calculates the average of integer values.

```java
double averageSalary =
        employees.stream()
                 .collect(
                     Collectors.averagingInt(
                         Employee::getSalary
                     )
                 );
```

Common use cases:

- Average salary

- Average marks

- Average ratings

---

# summarizingInt()

Produces multiple statistics in a single operation.

```java
IntSummaryStatistics statistics =
        employees.stream()
                 .collect(
                     Collectors.summarizingInt(
                         Employee::getSalary
                     )
                 );
```

Available statistics:

```java
statistics.getCount();

statistics.getSum();

statistics.getAverage();

statistics.getMin();

statistics.getMax();
```

Instead of performing multiple Stream operations, all statistics are calculated in one pass.

# Commonly Used Collectors

| Collector          | Purpose                  |
| ------------------ | ------------------------ |
| `toList()`         | Convert to List          |
| `toSet()`          | Convert to Set           |
| `toMap()`          | Convert to Map           |
| `joining()`        | Combine Strings          |
| `counting()`       | Count elements           |
| `groupingBy()`     | Group elements           |
| `partitioningBy()` | Split into two groups    |
| `mapping()`        | Transform grouped values |
| `summingInt()`     | Calculate sum            |
| `averagingInt()`   | Calculate average        |
| `summarizingInt()` | Calculate statistics     |

---

# Best Practices

- Prefer `groupingBy()` for categorizing data.

- Use `partitioningBy()` only for boolean conditions.

- Use `mapping()` to avoid collecting unnecessary objects.

- Use `summarizingInt()` instead of multiple aggregation operations.

- Keep collector expressions readable by formatting nested collectors.

---

# Common Mistakes

- Confusing `groupingBy()` with `partitioningBy()`.

- Using `toMap()` when duplicate keys may exist.

- Writing deeply nested collectors that reduce readability.

- Performing multiple aggregation operations when `summarizingInt()` is sufficient.

# Final Takeaway

> **`collect()` is the Stream API's primary terminal operation for accumulating results, while the `Collectors` utility class provides reusable strategies for converting, grouping, partitioning, transforming, and aggregating Stream data. Choosing the appropriate collector makes Stream pipelines both expressive and efficient.**
