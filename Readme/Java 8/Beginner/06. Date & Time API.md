# 6. Date & Time API

The **Java 8 Date & Time API** was introduced to replace the old date and time classes such as `Date`, `Calendar`, and `SimpleDateFormat`.

The new API is **immutable**, **thread-safe**, easier to use, and provides a much cleaner design for working with dates and times.

It is available in the `java.time` package.

---

# Why was the Date & Time API Introduced?

Before Java 8, Java primarily used:

- `Date`
- `Calendar`
- `SimpleDateFormat`

Although functional, these classes had several problems.

### Problems with the Old API

- Mutable objects
- `SimpleDateFormat` is not thread-safe
- Confusing API design
- Difficult date calculations
- Poor timezone handling

For example,

```java
Date date = new Date();

date.setYear(120);     // Confusing (2020)
date.setMonth(5);      // June (months start from 0)
```

Java 8 introduced a completely new Date & Time API to address these issues.

---

# Old API vs Java 8 Date & Time API

| Old API                | Java 8 API                 |
| ---------------------- | -------------------------- |
| Mutable                | Immutable                  |
| Not thread-safe        | Thread-safe                |
| Confusing              | Easy to read               |
| Difficult calculations | Rich utility methods       |
| Poor timezone support  | Excellent timezone support |

---

# Core Classes

The Java 8 Date & Time API contains several classes, each designed for a specific purpose.

| Class               | Represents               |
| ------------------- | ------------------------ |
| `LocalDate`         | Date only                |
| `LocalTime`         | Time only                |
| `LocalDateTime`     | Date and Time            |
| `ZonedDateTime`     | Date, Time and Timezone  |
| `Instant`           | Timestamp                |
| `Period`            | Difference between dates |
| `Duration`          | Difference between times |
| `DateTimeFormatter` | Formatting and Parsing   |

---

# LocalDate

`LocalDate` represents a **date without time or timezone**.

Use it whenever only the calendar date is required.

Examples include:

- Birthday
- Holiday
- Invoice Date
- Joining Date

---

## Creating LocalDate

```java
LocalDate today = LocalDate.now();

LocalDate birthday =
        LocalDate.of(1995, 5, 20);
```

---

## Common Methods

```java
today.plusDays(5);

today.minusMonths(1);

today.plusYears(2);

today.getYear();

today.getMonth();

today.getDayOfMonth();
```

---

# LocalTime

`LocalTime` represents a **time without date or timezone**.

Use it whenever only the time is required.

Examples include:

- Office opening time
- Alarm time
- Movie show time

---

## Creating LocalTime

```java
LocalTime now = LocalTime.now();

LocalTime meeting =
        LocalTime.of(10, 30);
```

---

## Common Methods

```java
now.plusHours(2);

now.minusMinutes(15);

now.getHour();

now.getMinute();
```

---

# LocalDateTime

`LocalDateTime` represents both **date and time** but **does not contain timezone information**.

Use it for local date-time values.

Examples include:

- Meeting schedule
- Order placed time
- Event start time

---

## Creating LocalDateTime

```java
LocalDateTime now =
        LocalDateTime.now();

LocalDateTime interview =
        LocalDateTime.of(
                2026,
                6,
                30,
                10,
                30
        );
```

---

## Common Methods

```java
now.plusDays(2);

now.minusHours(3);

now.getYear();

now.getMonth();

now.getHour();
```

---

# ZonedDateTime

`ZonedDateTime` represents a **date, time, and timezone**.

It is useful when applications work across multiple countries or time zones.

Examples include:

- International meetings
- Flight schedules
- Global applications

---

## Creating ZonedDateTime

```java
ZonedDateTime indiaTime =
        ZonedDateTime.now();

ZonedDateTime londonTime =
        ZonedDateTime.now(
                ZoneId.of("Europe/London")
        );
```

---

## Why is ZonedDateTime Important?

Suppose a meeting is scheduled for **10:00 AM IST**.

Developers in:

- India
- London
- New York

should all see the correct local time.

`ZonedDateTime` automatically handles timezone conversions.

---

# Real-world Usage

| Requirement           | Recommended Class |
| --------------------- | ----------------- |
| Birthday              | `LocalDate`       |
| Alarm Time            | `LocalTime`       |
| Order Date & Time     | `LocalDateTime`   |
| International Meeting | `ZonedDateTime`   |

---

# Key Points to Remember

- Use `LocalDate` for dates.
- Use `LocalTime` for times.
- Use `LocalDateTime` for local date and time.
- Use `ZonedDateTime` whenever timezones are involved.
- All Java 8 Date & Time classes are immutable.

--- 

# Instant

`Instant` represents a **specific moment on the timeline** (a timestamp).

Internally, it stores the number of seconds and nanoseconds since the Unix Epoch (`1970-01-01T00:00:00Z`).

It is commonly used for logging, auditing, distributed systems, and timestamps.

---

## Creating Instant

```java
Instant now = Instant.now();
```

---

## Common Methods

```java
now.plusSeconds(60);

now.minusSeconds(30);

now.getEpochSecond();
```

---

# Period

`Period` represents the difference between **two dates**.

It is date-based and works with:

- Years
- Months
- Days

Example

```java
LocalDate joiningDate =
        LocalDate.of(2020, 1, 15);

LocalDate today =
        LocalDate.now();

Period period =
        Period.between(joiningDate, today);

System.out.println(period.getYears());
```

Use `Period` whenever working with calendar dates.

Examples:

- Employee experience
- Age calculation
- Membership duration

---

# Duration

`Duration` represents the difference between **two times**.

It is time-based and works with:

- Hours
- Minutes
- Seconds
- Milliseconds
- Nanoseconds

Example

```java
LocalTime start =
        LocalTime.of(9, 0);

LocalTime end =
        LocalTime.of(17, 30);

Duration duration =
        Duration.between(start, end);

System.out.println(duration.toHours());
```

Use `Duration` whenever working with elapsed time.

Examples:

- Video length
- API response time
- Task execution time

---

# Period vs Duration

| Period                 | Duration                                           |
| ---------------------- | -------------------------------------------------- |
| Date-based             | Time-based                                         |
| Works with `LocalDate` | Works with `LocalTime`, `LocalDateTime`, `Instant` |
| Years, Months, Days    | Hours, Minutes, Seconds                            |
| Calendar calculations  | Time calculations                                  |

---

# DateTimeFormatter

`DateTimeFormatter` is used for formatting and parsing dates and times.

Unlike `SimpleDateFormat`, it is **immutable** and **thread-safe**.

---

## Formatting

```java
LocalDate today = LocalDate.now();

DateTimeFormatter formatter =
        DateTimeFormatter.ofPattern("dd-MM-yyyy");

String formatted =
        today.format(formatter);
```

Output

```
29-06-2026
```

---

## Parsing

```java
String date = "29-06-2026";

DateTimeFormatter formatter =
        DateTimeFormatter.ofPattern("dd-MM-yyyy");

LocalDate parsedDate =
        LocalDate.parse(date, formatter);
```

---

# Best Practices

- Prefer the Java 8 Date & Time API over the legacy `Date` and `Calendar` classes.
- Use the most appropriate class for the requirement (`LocalDate`, `LocalTime`, etc.).
- Use `ZonedDateTime` when working with multiple time zones.
- Use `Period` for date calculations and `Duration` for time calculations.
- Use `DateTimeFormatter` instead of `SimpleDateFormat`.

---

# Common Mistakes

- Using `LocalDateTime` when timezone information is required.
- Confusing `Period` with `Duration`.
- Continuing to use `Date` and `Calendar` for new applications.
- Using `SimpleDateFormat` in multi-threaded applications.

---

# Interview Questions

### 1. Why was the Java 8 Date & Time API introduced?

To replace the old mutable and confusing Date API with an immutable, thread-safe, and more intuitive API.

---

### 2. What is the difference between `LocalDate` and `LocalDateTime`?

`LocalDate` stores only the date.

`LocalDateTime` stores both the date and time.

---

### 3. When should you use `ZonedDateTime`?

When the application works across multiple time zones.

---

### 4. What is the difference between `Period` and `Duration`?

`Period` is used for date-based calculations.

`Duration` is used for time-based calculations.

---

### 5. Why is `DateTimeFormatter` preferred over `SimpleDateFormat`?

Because it is immutable and thread-safe.

---

### 6. What is an `Instant`?

An `Instant` represents a specific point in time (timestamp).
