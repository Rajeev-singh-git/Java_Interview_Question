# 3.2 Passing Behavior as a Parameter (The Java 8 Programming Model)

One of the biggest changes introduced in **Java 8** was the ability to write code that appears to **pass behavior as a parameter**.

For example,

```java
List<Integer> numbers = List.of(2, 5, 8, 11, 14);

numbers.stream()
       .filter(n -> n % 2 == 0)
       .forEach(System.out::println);
```

At first glance, it looks like the `filter()` method is accepting a Lambda Expression directly.

But what is **actually** being passed?

This chapter answers that question.

---

# The Problem

Suppose the Stream API wants to filter a collection.

The Stream knows **how to filter**, but it does **not know which condition** you want to apply.

For example,

* Keep only even numbers.
* Keep only adults.
* Keep only employees earning more than ₹50,000.

The filtering algorithm remains the same.

Only the **condition** changes.

So instead of hardcoding the condition, the Stream API asks the caller to provide it.

---

# Before Java 8

Before Java 8, Java could **not pass behavior directly**.

Developers typically used anonymous inner classes.

```java
Predicate<Integer> isEven = new Predicate<Integer>() {

    @Override
    public boolean test(Integer n) {
        return n % 2 == 0;
    }
};

numbers.stream().filter(isEven);
```

Although this works, it requires a significant amount of boilerplate code.

---

# Java 8 Solution

Java 8 introduced **Lambda Expressions** together with **Functional Interfaces**.

Now the same code becomes:

```java
numbers.stream()
       .filter(n -> n % 2 == 0);
```

Much shorter.

But something important is happening behind the scenes.

---

# What Does filter() Actually Accept?

The `filter()` method is defined approximately as:

```java
Stream<T> filter(Predicate<? super T> predicate);
```

Notice carefully.

`filter()` does **not** accept a Lambda Expression.

It accepts a **Predicate**.

```
filter(
    Predicate<T> predicate
)
```

So why does a Lambda work?

---

# The Compiler Does the Conversion

When the compiler sees:

```java
.filter(n -> n % 2 == 0)
```

it already knows that `filter()` expects a `Predicate`.

Therefore, it automatically converts the Lambda into an implementation of the `Predicate` Functional Interface.

Conceptually, the compiler treats it like this:

```java
Predicate<Integer> predicate = new Predicate<Integer>() {

    @Override
    public boolean test(Integer n) {
        return n % 2 == 0;
    }
};

numbers.stream().filter(predicate);
```

You never write this code manually.

The compiler generates the implementation for you.

---

# What Happens Inside filter()?

Once `filter()` receives the `Predicate`, it repeatedly calls its `test()` method.

Conceptually, it works like this:

```java
for (Integer n : numbers) {

    if (predicate.test(n)) {
        // keep the element
    }

}
```

For each element,

```
2

↓

predicate.test(2)

↓

true

↓

Keep
```

```
5

↓

predicate.test(5)

↓

false

↓

Discard
```

The Stream API never knows what your condition is.

It simply calls `predicate.test()`.

The implementation you supplied decides whether an element is accepted.

---

# What Is Actually Being Passed?

Many developers say:

> "Java 8 allows us to pass methods as parameters."

This is **not entirely accurate**.

Java still does **not** pass methods directly.

Instead, Java passes an **object** that implements a Functional Interface.

```
Method

↓

Cannot be passed directly
```

Instead,

```
Lambda Expression

↓

Compiler creates

↓

Functional Interface Implementation

↓

Passed as a parameter
```

So the real idea is:

> **Java does not pass methods. It passes objects that represent behavior.**

---

# The Java 8 Programming Model

Modern Java APIs follow the same design pattern.

The API defines **what kind of behavior it needs** by accepting a Functional Interface.

The caller supplies the behavior using a Lambda Expression or Method Reference.

```
API Method
        │
expects
        ▼
Functional Interface
        │
Defines Contract
        ▼
Lambda Expression
        │
Provides Implementation
        ▼
Compiler
        │
Creates Functional Interface Object
        ▼
API Executes It
```

# More Examples

## Stream.filter()

```java
stream.filter(n -> n % 2 == 0);
```

The `filter()` method expects a `Predicate`.

---

## Stream.map()

```java
stream.map(String::length);
```

The `map()` method expects a `Function`.

---

## forEach()

```java
list.forEach(System.out::println);
```

The `forEach()` method expects a `Consumer`.

---

## CompletableFuture

```java
future.thenApply(String::length);
```

`thenApply()` expects a `Function`.

---

Although the APIs are different, the programming model is exactly the same.

# Mental Model

Whenever you see a Lambda Expression being passed to a method, think:

```
Method

↓

Expects Functional Interface

↓

Lambda implements Functional Interface

↓

Compiler creates object

↓

Method executes the SAM
```

---

# Why This Design Is Powerful

Instead of writing a new method for every condition, transformation, or action, Java APIs simply ask the caller to provide the required behavior.

This makes APIs:

* More reusable
* More flexible
* Easier to extend
* Easier to compose

The algorithm remains the same.

Only the behavior changes.

---

# Common Interview Traps

* Java does **not** pass methods directly.
* A Lambda Expression is **not** a Functional Interface.
* The compiler converts the Lambda into an implementation of the required Functional Interface.
* The receiving method calls the Functional Interface's Single Abstract Method (SAM).

---

# Interview Questions

### 1. Does Java pass methods as parameters?

Not directly. Java passes an implementation of a Functional Interface. Lambda Expressions and Method References provide a concise way to create that implementation, giving the effect of passing behavior as a parameter.



---

### 2. Why can `filter()` accept a Lambda Expression?

Because `filter()` expects a `Predicate`, and the compiler automatically converts the Lambda into a `Predicate` implementation.

`filter()` does not accept a Lambda Expression directly. It accepts a `Predicate`. Since `Predicate` is a Functional Interface, the compiler automatically converts the Lambda Expression into a `Predicate` implementation. This allows Java to pass behavior as a parameter through Functional Interface objects.

---

### 3. What is actually passed to `filter()`?

A `Predicate` object.

---

### 4. Who creates the `Predicate` object?

The Java compiler and JVM create it from the Lambda Expression.

---

### 5. How does `filter()` execute the Lambda?

It calls the `test()` method of the supplied `Predicate`.
