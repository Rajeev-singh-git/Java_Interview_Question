# 3.4 Function

`Function<T, R>` is one of the most commonly used built-in Functional Interfaces in Java.

It represents a **transformation** that accepts one input and produces one output.

Unlike `Predicate`, which answers a **yes/no question**, `Function` transforms data from one form into another.

It is heavily used in the **Stream API**, **CompletableFuture**, **DTO mapping**, and many modern Java frameworks.

---

# Why was Function introduced?

Many applications need to transform one object into another.

Examples include:

- Convert a String to uppercase.
- Find the length of a String.
- Convert an `Employee` object into an `EmployeeDTO`.
- Extract the salary from an Employee.
- Convert a price from USD to INR.

Before Java 8, developers usually created separate methods for every transformation.

```java
public int getLength(String str) {
    return str.length();
}
```

or anonymous inner classes.

Java 8 introduced `Function` so that transformations could be treated as reusable behavior and passed wherever needed.

---

# What Problem Does Function Solve?

Suppose we have an `Employee`.

Sometimes we need:

- Employee → Salary
- Employee → Name
- Employee → Department
- Employee → EmployeeDTO

Instead of creating separate methods everywhere, Java allows us to pass the transformation itself.

```java
employee -> employee.getSalary()
```

The API simply applies whatever transformation you provide.

# Predicate vs Function

A common interview mistake is confusing `Predicate` with `Function`.

| Predicate          | Function         |
| ------------------ | ---------------- |
| Checks a condition | Transforms data  |
| Returns `boolean`  | Returns any type |
| Used for filtering | Used for mapping |

Examples

Predicate

```java
age -> age >= 18
```

returns

```
true / false
```

Function

```java
name -> name.length()
```

returns

```
Integer
```

# Definition

`Function<T, R>` is a built-in Functional Interface available in the `java.util.function` package.

It accepts one input of type **T** and returns one output of type **R**.

---

# Method Signature

```java
@FunctionalInterface
public interface Function<T, R> {

    R apply(T t);

}
```

```

↓

↓

```

Meaning

```

↓

↓

```

# The apply() Method

The `apply()` method performs the transformation.

Syntax

```java
R apply(T t);
```

Example

```java
Function<String, Integer> length = str -> str.length();

System.out.println(length.apply("Java"));
```

Output

```
4
```

---

```java
Function<String, Integer> length =

System.out.println(length.apply("Hello"));
```

Output

```
5
```

## Example 2 : Square a Number

# Built-in Methods

Besides `apply()`, `Function` provides several useful methods for combining transformations.

---

## 1. andThen()

Executes the current Function first, then applies another Function to the result.

```java
Input

↓

Function A

↓

Function B

↓

Final Output
```

Example

```java
Function<String, String> trim =
        String::trim;

Function<String, String> upper =
        String::toUpperCase;

Function<String, String> result =
        trim.andThen(upper);

System.out.println(result.apply(" java "));
```

Output

```java
JAVA
```

---

## 2. compose()

Executes another Function first, then applies the current Function.

```java
Input

↓

Function B

↓

Function A

↓

Final Output
```

Example

```java
Function<String, String> trim =
        String::trim;

Function<String, String> upper =
        String::toUpperCase;

Function<String, String> result =
        upper.compose(trim);

System.out.println(result.apply(" java "));
```

Output

```java
JAVA
```

---

## compose() vs andThen()

Suppose

```java
A = trim()

B = toUpperCase()
```

Using

```java
A.andThen(B)
```

Execution

```java
  trim()

    ↓

toUpperCase()
```

Using

```java
A.compose(B)
```

Execution

```java
toUpperCase()

↓

trim()
```

A simple way to remember:

- **andThen()** → Current Function **then** next Function.
- **compose()** → Apply the **other Function first**, then the current Function.

---

## 3. identity()

Returns a Function that simply returns its input without modifying it.

```java
Function<String, String> identity =
        Function.identity();

System.out.println(identity.apply("Java"));
```

Output

```java
Java
```

This is useful when no transformation is required but a Function is still expected.

---

# Real-world Usage

`Function` is one of the most frequently used Functional Interfaces in modern Java applications.

Whenever data needs to be **transformed** from one form to another, `Function` is usually involved.

Common production use cases include:

- Stream API (`map()`)
- Entity → DTO conversion
- DTO → Entity conversion
- Data formatting
- JSON transformation
- CompletableFuture (`thenApply()`)
- Spring Framework APIs

---

## Stream API

The `map()` method expects a `Function`.

```java
List<String> names = List.of("Alice", "Bob", "Charlie");

names.stream()
     .map(String::length)
     .forEach(System.out::println);
```

Output

```java
5
3
7
```

Here,

- `map()` expects a `Function`
- The Function transforms a `String` into an `Integer`

---

## Entity to DTO Mapping

One of the most common backend use cases.

```java
Function<Employee, EmployeeDTO> mapper =
        emp -> new EmployeeDTO(emp.getId(), emp.getName());
```

Instead of exposing database entities directly, applications often transform them into DTOs before sending responses.

---

## Data Formatting

```java
Function<Double, String> formatSalary =
        salary -> "₹" + salary;
```

Transforms a numeric value into a formatted string.

---

## CompletableFuture

```java
future.thenApply(String::length);
```

`thenApply()` expects a `Function` to transform the previous result.

---

# Best Practices

- Keep Functions focused on a single transformation.
- Use Method References whenever they improve readability.
- Prefer small reusable Functions over large complex ones.
- Use `compose()` and `andThen()` to build transformation pipelines.
- Give Functions meaningful names such as `toUpperCase`, `extractSalary`, or `entityToDto`.

---

# Common Mistakes

- Using `Function` when only a condition is required (use `Predicate` instead).
- Using `Function` when no value is returned (use `Consumer` instead).
- Confusing `compose()` with `andThen()`.
- Writing very large transformation logic inside a single Function.

---

# Common Interview Traps

- `Function` always accepts one input and returns one output.
- `apply()` is the Single Abstract Method (SAM).
- `compose()` executes the supplied Function first.
- `andThen()` executes the current Function first.
- `identity()` returns the input unchanged.
- `map()` in the Stream API expects a `Function`.

# Interview Scenario / Real-world Motivation

**Scenario: Building a REST API**

Suppose your application retrieves an `Employee` entity from the database.

```java
Employee employee = employeeRepository.findById(id);
```

However, exposing the entity directly is not recommended.

Instead, you convert it into an `EmployeeDTO`.

```java
Function<Employee, EmployeeDTO> mapper =
        emp -> new EmployeeDTO(emp.getId(), emp.getName());

EmployeeDTO dto = mapper.apply(employee);
```

This transformation pattern is used extensively in Spring Boot applications.

Similarly, Stream API's `map()` performs the same kind of transformation on every element in a collection.

Understanding `Function` helps explain why data mapping is so common in modern backend development.

# What's Next?

A `Function` transforms one value into another.

But what if we don't want to return anything?

Sometimes we simply want to **perform an action**, such as:

- Printing a value
- Sending an email
- Logging information
- Saving data

That's where the **Consumer** Functional Interface comes in.
