# 3.3 Predicate

`Predicate<T>` is one of Java's most commonly used built-in Functional Interfaces.

It represents a **condition** that takes one input and returns either **`true`** or **`false`**.

Predicate is widely used for **filtering**, **validation**, **searching**, and **decision-making** in modern Java applications.

---

## Why was Predicate introduced?

Many applications need to answer simple yes/no questions.

Examples include:

- Is the employee eligible for a bonus?
- Is the user an adult?
- Is the product in stock?
- Is the number even?
- Is the email valid?

Before Java 8, these conditions were usually written as separate methods or anonymous inner classes.

Java 8 introduced **Predicate** to represent such conditions as reusable objects that can be passed to methods.

---

## What problem does Predicate solve?

Suppose we want to filter only adult users.

Without Predicate

```java
public boolean isAdult(Person person) {
    return person.getAge() >= 18;
}
```

Now imagine several APIs needing different conditions.

Instead of creating multiple methods, we can simply pass the condition itself.

```java
person -> person.getAge() >= 18
```

Predicate makes conditions reusable, composable, and easy to pass around.

---

# Mental Model

Think of a Predicate as asking a **Yes/No question**.

```java
Input
   │
   ▼
Ask Question
   │
   ▼
true / false
```

Examples

```java
Age >= 18 ?

↓

true
```

```java
Salary > 50000 ?

↓

false
```

Whenever your logic answers **only true or false**, think **Predicate**.

---

# Definition

`Predicate<T>` is a built-in Functional Interface present in the `java.util.function` package.

It accepts one input and returns a boolean result.

---

# Method Signature

```java
@FunctionalInterface
public interface Predicate<T> {

    boolean test(T t);
}
```

```java
T
│
▼
Predicate
│
▼

```

Meaning

```java
Input

↓

Condition

↓

true / false
```

# The test() Method

The `test()` method evaluates the given condition.

Syntax

```java
boolean test(T t);
```

Example

```java
Predicate<Integer> isEven = n -> n % 2 == 0;

System.out.println(isEven.test(10));
```

Output

```java
true
```

# Built-in Methods

Besides `test()`, Predicate provides several useful default methods for combining conditions.

---

## 1. and()

Combines two Predicates using logical AND.

Both conditions must be true.

```java
Predicate<Integer> greaterThan10 = n -> n > 10;
Predicate<Integer> even = n -> n % 2 == 0;

Predicate<Integer> result =
        greaterThan10.and(even);

System.out.println(result.test(20));
```

Output

```java
true
```

---

## 2. or()

Combines two Predicates using logical OR.

At least one condition must be true.

```java
Predicate<Integer> lessThan10 = n -> n < 10;
Predicate<Integer> even = n -> n % 2 == 0;

Predicate<Integer> result =
        lessThan10.or(even);

System.out.println(result.test(7));
```

Output

```java
true
```

---

## 3. negate()

Reverses the condition.

```java
Predicate<Integer> even = n -> n % 2 == 0;

Predicate<Integer> odd =
        even.negate();

System.out.println(odd.test(7));
```

Output

```java
true
```

---

## 4. isEqual()

Returns a Predicate that checks equality.

```java
Predicate<String> check =
        Predicate.isEqual("Java");

System.out.println(check.test("Java"));
```

Output

```java
true
```

---

# Predicate Chaining

Multiple conditions can be combined to build complex business rules.

```java
Predicate<Employee> experienced =
        emp -> emp.getExperience() >= 5;

Predicate<Employee> highSalary =
        emp -> emp.getSalary() >= 100000;

Predicate<Employee> eligible =
        experienced.and(highSalary);
```

Instead of writing one huge condition, Predicate allows composing smaller reusable conditions.

```java
Condition 1

AND

Condition 2

↓

Final Condition 
```

---
# Real-world Usage

`Predicate` is widely used wherever an application needs to evaluate a condition.

Common production use cases include:

- Filtering collections
- Stream API (`filter()`)
- Input validation
- Employee eligibility checks
- Product availability checks
- Permission and authorization rules
- Business rule validation

Example: Filter only even numbers.

```java
List<Integer> numbers = List.of(2, 5, 8, 11, 14);

numbers.stream()
       .filter(n -> n % 2 == 0)
       .forEach(System.out::println);
```

Output

```
2
8
14
```

Here,

- `filter()` expects a `Predicate`
- The Lambda provides the condition

Whenever an API needs to answer **"Should this object be accepted?"**, it usually expects a `Predicate`.

---

# Best Practices

- Keep Predicates focused on a single condition.
- Combine small Predicates using `and()` and `or()` instead of writing long conditions.
- Reuse Predicates whenever possible.
- Prefer meaningful variable names such as `isAdult`, `isEligible`, or `hasPermission`.
- Use Method References when they improve readability.

---

# Common Mistakes

- Using `Predicate` when a value needs to be returned (use `Function` instead).
- Writing very complex conditions inside a single Predicate.
- Forgetting that `Predicate` always returns a boolean.
- Creating duplicate Predicates instead of reusing existing ones.

---

# Common Interview Traps

- `Predicate` always returns `boolean`.
- `test()` is the Single Abstract Method (SAM).
- `and()` and `or()` return new Predicates—they do not modify existing ones.
- `negate()` reverses the result of a Predicate.
- `Predicate` is commonly used with Stream API's `filter()`.

# Interview Questions

### 1. What is Predicate?

`Predicate<T>` is a Functional Interface that accepts one input and returns a boolean result.

---

### 2. When should Predicate be used?

Whenever a condition needs to be evaluated and the result is either **true** or **false**.

---

### 3. What is the SAM of Predicate?

```java
boolean test(T t);
```

---

### 4. What is the purpose of `test()`?

It evaluates the condition and returns `true` or `false`.

---

### 5. What is the difference between Predicate and Function?

| Predicate | Function |
|-----------|----------|
| Returns `boolean` | Returns any type |
| Used for conditions | Used for transformations |

---

### 6. What does `and()` do?

Combines two Predicates using logical **AND**.

Both conditions must evaluate to `true`.

---

### 7. What does `or()` do?

Combines two Predicates using logical **OR**.

At least one condition must evaluate to `true`.

---

### 8. What does `negate()` do?

Returns the opposite result of a Predicate.

Example:

```
Even

↓

Negate

↓

Odd
```

---

### 9. What is `Predicate.isEqual()`?

It creates a Predicate that checks whether an input is equal to a specified value.

---

### 10. Where is Predicate commonly used?

- Stream API
- Collection filtering
- Validation
- Business rules
- Authorization
- Search operations

---

### 11. Can Predicates be chained?

Yes.

Using

- `and()`
- `or()`
- `negate()`

---

### 12. Why is Predicate useful?

It makes conditions reusable, composable, and easy to pass to APIs.
