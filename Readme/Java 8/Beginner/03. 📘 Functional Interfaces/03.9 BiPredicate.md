# 3.9 BiPredicate

`BiPredicate<T, U>` is a built-in Functional Interface that represents a **condition** involving **two input values**.

It is the **two-input version of `Predicate`**.

It accepts two inputs and returns either **`true`** or **`false`**.

Whenever a condition depends on two different values, `BiPredicate` is usually the appropriate choice.

---

# Why was BiPredicate introduced?

Many business rules require comparing or validating two values.

Examples include:

- Are the username and password valid?
- Does an employee belong to a department?
- Is the entered password equal to the stored password?
- Are two strings equal ignoring case?
- Does an order belong to a customer?

Although these operations can be implemented using custom methods, Java provides `BiPredicate` to represent reusable conditions involving **two inputs**.

---

# What Problem Does BiPredicate Solve?

Suppose we want to compare two passwords.

Without `BiPredicate`

```java
public boolean isValid(String entered, String stored) {
    return entered.equals(stored);
}
```

Using `BiPredicate`

```java
BiPredicate<String, String> validate =
        String::equals;
```

The validation logic becomes reusable and can be passed to APIs expecting a `BiPredicate`.

# Definition

`BiPredicate<T, U>` is a built-in Functional Interface available in the `java.util.function` package.

It is the **two-input version of `Predicate`**.

It accepts two inputs and returns a boolean result.

---

# Method Signature

```java
@FunctionalInterface
public interface BiPredicate<T, U> {

    boolean test(T t, U u);

}
```

Meaning

```java
Input 1

Input 2

↓

Evaluate Condition

↓

boolean
```

---

# The test() Method

The `test()` method evaluates the supplied condition and returns either `true` or `false`.

Syntax

```java
boolean test(T t, U u);
```

---

# Examples

## Example 1 : Compare Two Numbers

```java
BiPredicate<Integer, Integer> greater =
        (a, b) -> a > b;

System.out.println(greater.test(20, 10));
```

Output

```
true
```

---

## Example 2 : Compare Two Strings

```java
BiPredicate<String, String> equal =
        String::equals;

System.out.println(equal.test("Java", "Java"));
```

Output

```
true
```

---

## Example 3 : Validate Login

```java
BiPredicate<String, String> login =
        (enteredPassword, storedPassword) ->
                enteredPassword.equals(storedPassword);
```

---

## Example 4 : Employee Department Check

```java
BiPredicate<Employee, String> belongsToDepartment =
        (employee, department) ->
                employee.getDepartment().equals(department);
```

# Built-in Methods

Like `Predicate`, `BiPredicate` provides several useful default methods.

---

## and()

Returns `true` only if **both** BiPredicates return `true`.

```java
BiPredicate<Integer, Integer> positive =
        (a, b) -> a > 0 && b > 0;

BiPredicate<Integer, Integer> even =
        (a, b) -> a % 2 == 0 && b % 2 == 0;

BiPredicate<Integer, Integer> check =
        positive.and(even);

System.out.println(check.test(10, 20));
```

---

## or()

Returns `true` if **at least one** BiPredicate returns `true`.

---

## negate()

Returns the opposite result of the original condition.

---

# Real-world Usage

`BiPredicate` is commonly used whenever a decision depends on two values.

Common production use cases include:

- Login validation
- Password verification
- Comparing two objects
- Permission checks
- Business rule validation
- Relationship validation

---

# Best Practices

- Keep each BiPredicate focused on a single condition.
- Use meaningful names such as `isValidLogin` or `belongsToDepartment`.
- Combine simple BiPredicates using `and()` and `or()`.
- Prefer Method References whenever they improve readability.

---

# Common Mistakes

- Using `BiPredicate` when only one input is required.
- Writing very complex conditions inside `test()`.
- Forgetting that `test()` always returns a boolean.
- Using `BiPredicate` when a transformation is actually required.

# Final Takeaway

> **Whenever your logic evaluates a condition using two input values, think `BiPredicate`. It is the two-input version of `Predicate`, allowing a boolean condition to be evaluated using two inputs instead of one.**

---

# Key Points to Remember

- `BiPredicate<T, U>` is the two-input version of `Predicate`.
- It accepts two inputs.
- It always returns a boolean.
- `test()` is its Single Abstract Method.
- It supports `and()`, `or()`, and `negate()`.
- It is commonly used for validation, comparison, and business rules involving two values.
