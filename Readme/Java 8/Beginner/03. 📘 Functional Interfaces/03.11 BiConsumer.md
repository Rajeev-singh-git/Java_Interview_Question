# 3.11 BiConsumer

`BiConsumer<T, U>` is a built-in Functional Interface that represents an **operation** involving **two input values**.

It is the **two-input version of `Consumer`**.

It accepts two inputs and **does not return any result**.

Whenever an operation needs to perform an action using two different values, `BiConsumer` is usually the appropriate choice.

---

# Why was BiConsumer introduced?

Many business operations require two input values to perform an action.

Examples include:

- Print a key and its value.
- Save an employee with a department.
- Process an order for a customer.
- Log a username with a timestamp.
- Send a notification to a specific user.

Although these operations can be implemented using custom methods, Java provides `BiConsumer` to represent reusable actions involving **two inputs**.

---

# What Problem Does BiConsumer Solve?

Suppose we want to print a student's name and marks.

Without `BiConsumer`

```java
public void print(String name, Integer marks) {
    System.out.println(name + " : " + marks);
}
```

Using `BiConsumer`

```java
BiConsumer<String, Integer> print =
        (name, marks) ->
                System.out.println(name + " : " + marks);
```

The action becomes reusable and can be passed wherever a `BiConsumer` is expected.

# Definition

`BiConsumer<T, U>` is a built-in Functional Interface available in the `java.util.function` package.

It is the **two-input version of `Consumer`**.

It accepts two inputs and returns nothing.

---

# Method Signature

```java
@FunctionalInterface
public interface BiConsumer<T, U> {

    void accept(T t, U u);

}
```

Meaning

```java
Input 1

Input 2

↓

Perform Action

↓

void
```

---

# The accept() Method

The `accept()` method performs the required action using the supplied inputs.

Syntax

```java
void accept(T t, U u);
```

# Examples

## Example 1 : Print Student Details

```java
BiConsumer<String, Integer> print =
        (name, marks) ->
                System.out.println(name + " : " + marks);

print.accept("Alice", 95);
```

Output

```
Alice : 95
```

---

## Example 2 : Save Employee

```java
BiConsumer<Employee, String> assignDepartment =
        (employee, department) ->
                employee.setDepartment(department);
```

---

## Example 3 : Print Map Entries

```java
Map<String, Integer> marks = Map.of(
        "Alice", 95,
        "Bob", 88
);

marks.forEach(
        (name, score) ->
                System.out.println(name + " : " + score)
);
```

Output

```java
Alice : 95
Bob : 88
```

---

## Example 4 : Log User Activity

```java
BiConsumer<String, LocalDateTime> logger =
        (user, time) ->
                System.out.println(user + " logged in at " + time);
```

# Built-in Methods

Like `Consumer`, `BiConsumer` provides one useful default method.

---

## andThen()

Executes the current `BiConsumer` first and then executes another `BiConsumer` using the same inputs.

```java
BiConsumer<String, Integer> print =
        (name, marks) ->
                System.out.println(name + " : " + marks);

BiConsumer<String, Integer> congratulate =
        (name, marks) -> {
            if (marks >= 90) {
                System.out.println("Excellent!");
            }
        };

BiConsumer<String, Integer> process =
        print.andThen(congratulate);

process.accept("Alice", 95);
```

Output

```java
Alice : 95
Excellent!
```

# Real-world Usage

`BiConsumer` is commonly used whenever an action needs to be performed using two values.

Common production use cases include:

- Processing `Map` entries using `Map.forEach()`
- Logging information
- Updating objects
- Saving related data
- Event handling
- Processing key-value pairs

---

# Best Practices

- Keep each `BiConsumer` focused on a single responsibility.
- Prefer Method References whenever they improve readability.
- Use meaningful names such as `printMarks`, `assignDepartment`, or `logActivity`.
- Use `andThen()` to combine related actions.

---

# Common Mistakes

- Using `BiConsumer` when a result should be returned.
- Writing complex business logic inside `accept()`.
- Forgetting that `accept()` always returns `void`.
- Using `BiConsumer` when only one input is required.

# Interview Scenario / Real-world Motivation

### Scenario: Processing Employee Departments

Suppose an HR application receives an employee and the department to which the employee should be assigned.

Instead of writing assignment logic repeatedly, we can define a reusable `BiConsumer`.

```java
BiConsumer<Employee, String> assignDepartment =
        Employee::setDepartment;
```

Whenever an employee needs to be assigned to a department,

```java
assignDepartment.accept(employee, "Engineering");
```

The same action can now be reused throughout the application.

# Key Points to Remember

- `BiConsumer<T, U>` is the two-input version of `Consumer`.
- It accepts two inputs.
- It always returns `void`.
- `accept()` is its Single Abstract Method.
- `andThen()` chains multiple actions.
