# 3.1 Functional Interfaces

Functional Interfaces are one of the core concepts introduced in **Java 8** to enable **Functional Programming**.

A **Lambda Expression** by itself cannot exist—it always targets a **Functional Interface**. Functional Interfaces act as the **contract** that tells the compiler **which method a Lambda Expression should implement**.

Modern Java features such as **Lambda Expressions**, **Method References**, **Streams**, **Optional**, and **CompletableFuture** all rely on Functional Interfaces.

---

# Why were Functional Interfaces introduced?

Before Java 8, behavior was typically passed using **anonymous inner classes**.

```java
Runnable r = new Runnable() {
    @Override
    public void run() {
        System.out.println("Hello");
    }
};
```

Although effective, this approach required a significant amount of boilerplate code.

Java 8 introduced **Lambda Expressions** to make code concise.

However, a Lambda Expression contains only the **implementation**.

It does **not** specify **which method** it belongs to.

The compiler needs a target method to attach the Lambda Expression to.

Functional Interfaces provide that target by exposing exactly **one abstract method**.

```
Lambda Expression
        │
Provides Implementation
        │
        ▼
Functional Interface
        │
Defines Contract (SAM)
```

Without Functional Interfaces, Lambda Expressions would have no method to implement.

---

# What is a Functional Interface?

A Functional Interface is an interface that contains **exactly one abstract method**.

It may also contain:

- Default methods
- Static methods
- Private methods *(Java 9+)*

Only the number of **abstract methods** matters.

---

# Mental Model

Think of a Functional Interface as a **contract**.

The contract defines **what needs to be implemented**.

The Lambda Expression simply provides **how it is implemented**.

```
Functional Interface
        │
        ▼
     Contract
        │
        ▼
Lambda Expression
        │
        ▼
Implementation
```

Without the contract, the compiler would not know which method the Lambda Expression is implementing.

---

# Single Abstract Method (SAM)

A Functional Interface is also known as a **SAM (Single Abstract Method) Interface** because it contains exactly one abstract method.

```java
@FunctionalInterface
interface Greeting {

    void greet();
}
```

Here,

```java
void greet();
```

is the **Single Abstract Method (SAM)**.

A Lambda Expression provides the implementation of this method.

---

# @FunctionalInterface Annotation

`@FunctionalInterface` is an optional annotation introduced in Java 8.

It instructs the compiler to verify that the interface always remains a Functional Interface.

```java
@FunctionalInterface
interface Calculator {

    int add(int a, int b);
}
```

If another abstract method is added, the compiler reports a compilation error.

```java
@FunctionalInterface
interface Calculator {

    int add(int a, int b);

    int subtract(int a, int b);   // Compilation Error
}
```

Although optional, using `@FunctionalInterface` is considered a **best practice** because it provides compile-time validation.

---

# Default and Static Methods

A Functional Interface may contain any number of **default** and **static** methods.

Only the number of **abstract methods** matters.

```java
@FunctionalInterface
interface Calculator {

    int add(int a, int b);

    default void print() {
        System.out.println("Calculator");
    }

    static void info() {
        System.out.println("Functional Interface");
    }
}
```

This is still a Functional Interface because it contains only **one abstract method**.

```
Abstract Methods      → Exactly One ✅

Default Methods       → Unlimited ✅

Static Methods        → Unlimited ✅

Private Methods       → Allowed (Java 9+) ✅
```

---

# Functional Interfaces and Inheritance

A Functional Interface can extend another interface **as long as the resulting interface still contains exactly one abstract method**.

```java
@FunctionalInterface
interface A {

    void show();
}

@FunctionalInterface
interface B extends A {

}
```

`B` is still a Functional Interface because it inherits only one abstract method.

However,

```java
@FunctionalInterface
interface A {

    void show();
}

interface B {

    void print();
}

@FunctionalInterface
interface C extends A, B {

}
```

This is **not** a Functional Interface because `C` now contains **two abstract methods**.

---

# Custom Functional Interfaces

Java provides many built-in Functional Interfaces such as `Predicate`, `Function`, `Consumer`, and `Supplier`.

If none of them satisfy your requirements, you can create your own.

```java
@FunctionalInterface
interface Calculator {

    int add(int a, int b);
}
```

Whenever possible, prefer Java's built-in Functional Interfaces over creating custom ones, as they improve readability and consistency across projects.

---

# Relationship with Lambda Expressions

A Functional Interface and a Lambda Expression are **not the same thing**.

A Functional Interface defines **what needs to be implemented**, while the Lambda Expression provides **the implementation**.

```
Lambda Expression
        │
Provides Implementation
        │
        ▼
Functional Interface
        │
Defines Contract (SAM)
```

Think of it this way:

- **Functional Interface = Contract**
- **Lambda Expression = Implementation**

Method References follow the same principle and will be covered in the next chapter.

# Real-world Usage

Functional Interfaces are the foundation of modern Java APIs that accept **behavior as an argument**.

Whenever you see code that passes a Lambda Expression or Method Reference, a Functional Interface is usually involved.

Common production use cases include:

- Stream API (`Predicate`, `Function`, `Consumer`)
- Collections Framework (`Comparator`)
- CompletableFuture callbacks
- ExecutorService (`Runnable`, `Callable`)
- Event handling
- Spring Framework APIs

For example, the `filter()` method of the Stream API accepts a `Predicate`.

```java
List<String> names = List.of("Alice", "Bob", "Charlie");

names.stream()
     .filter(name -> name.length() > 3)
     .forEach(System.out::println);
```

Here,

- `Predicate<String>` defines the contract.
- The Lambda Expression provides its implementation.

---

# Best Practices

- Always use `@FunctionalInterface` for custom Functional Interfaces.
- Prefer Java's built-in Functional Interfaces whenever possible.
- Create custom Functional Interfaces only when the built-in ones don't meet your requirements.
- Keep Functional Interfaces focused on a single responsibility.
- Give custom Functional Interfaces meaningful names.

---

# Common Mistakes

- Adding more than one abstract method.
- Creating custom Functional Interfaces unnecessarily.
- Confusing Functional Interfaces with Lambda Expressions.
- Forgetting that default and static methods do **not** affect whether an interface is functional.

---

# Common Interview Traps

- A Functional Interface may contain many `default` and `static` methods.
- Only **one abstract method** is allowed.
- `@FunctionalInterface` is optional but recommended.
- A Lambda Expression is **not** a Functional Interface.
- Every Functional Interface is **not necessarily** annotated with `@FunctionalInterface`.

# Interview Questions

### 1. What is a Functional Interface?

A Functional Interface is an interface containing exactly one abstract method.

---

### 2. Why are Functional Interfaces required for Lambda Expressions?

A Lambda Expression only provides an implementation.

A Functional Interface provides the target type and the Single Abstract Method (SAM) that the Lambda implements.

---

### 3. What is a Single Abstract Method (SAM)?

The single abstract method present in a Functional Interface.

A Lambda Expression provides the implementation of this method.

---

### 4. Can a Functional Interface contain default and static methods?

Yes.

Only the number of abstract methods matters.

---

### 5. Is `@FunctionalInterface` mandatory?

No.

It is optional but recommended because it enables compile-time validation.

---

### 6. Can a Functional Interface extend another interface?

Yes.

As long as the resulting interface still contains exactly one abstract method.

---

### 7. Can an interface having two abstract methods be a Functional Interface?

No.

A Functional Interface must contain exactly one abstract method.

---

### 8. What is the difference between a Functional Interface and a Lambda Expression?

A Functional Interface defines the contract.

A Lambda Expression provides the implementation.

---

### 9. Why should built-in Functional Interfaces be preferred over custom ones?

Because they improve readability, consistency, and interoperability with Java APIs.

---

### 10. Does `Object` class methods affect whether an interface is functional?

No.

Methods inherited from `Object` (such as `toString()`, `equals()`, and `hashCode()`) are **not counted** when determining whether an interface is a Functional Interface.

---

### 11. Can a Functional Interface contain private methods?

Yes.

Private methods are allowed from Java 9 onwards and do not affect the Functional Interface rule.

---

### 12. Can an interface without the `@FunctionalInterface` annotation still be a Functional Interface?

Yes.

The annotation is optional.

The compiler determines whether an interface is functional based on the number of abstract methods.
