# 3.8 BinaryOperator

`BinaryOperator<T>` is a specialized Functional Interface that represents an operation on **two operands** where the **input and output are of the same type**.

It is a specialization of `BiFunction<T, T, T>`.

Whenever an operation combines two values of the same type and produces a result of the same type, `BinaryOperator` is usually the best choice.

It is the **two-input version of `Function` where both inputs and the output are the same type**.

More specifically, it is a specialization of **`BiFunction<T, T, T>`**.

---

# Why was BinaryOperator introduced?

Many operations combine two values of the same type.

Examples include:

- Add two numbers.
- Find the larger of two values.
- Find the smaller of two values.
- Concatenate two Strings.
- Merge two objects.

Although these operations can be implemented using `BiFunction<T, T, T>`, Java provides `BinaryOperator` because it clearly expresses that both inputs and the output are of the same type.

---

# What Problem Does BinaryOperator Solve?

Suppose we want to add two integers.

Using `BiFunction`

```java
BiFunction<Integer, Integer, Integer> add =
        (a, b) -> a + b;
```

All three types are `Integer`.

Instead of repeating the same type three times, Java provides `BinaryOperator`.

```java
BinaryOperator<Integer> add =
        (a, b) -> a + b;
```

The code becomes shorter and easier to understand.

---

```java

        │

```

Examples


```java

```

```java

↓

```

Whenever your logic combines two values into one value of the same type, think **BinaryOperator**.

---

# Definition

`BinaryOperator<T>` is a built-in Functional Interface available in the `java.util.function` package.

It accepts two inputs of the same type and returns the same type.

It extends `BiFunction<T, T, T>`.

---

# Method Signature

```java
@FunctionalInterface
public interface BinaryOperator<T>
        extends BiFunction<T, T, T> {
}
```

Since it extends `BiFunction`, its Single Abstract Method is:

```java
T apply(T t1, T t2);
```

---

# Examples

## Example 1 : Add Two Numbers

```java
BinaryOperator<Integer> add =
        (a, b) -> a + b;

System.out.println(add.apply(10, 20));
```

Output

```java
30
```

---

## Example 2 : Multiply Two Numbers

```java
BinaryOperator<Integer> multiply =

System.out.println(multiply.apply(5, 4));
```

Output

```
20
```

---

## Example 3 : Concatenate Strings

```java
BinaryOperator<String> concatenate =

```

# Built-in Methods

Besides the methods inherited from `BiFunction`, `BinaryOperator` provides two useful static methods.

---

## maxBy()

Returns the larger of two values based on the provided `Comparator`.

Example

```java
BinaryOperator<Integer> max =
        BinaryOperator.maxBy(Integer::compareTo);

System.out.println(max.apply(10, 25));
```

Output

```java
25
```

---

## minBy()

Returns the smaller of two values based on the provided `Comparator`.

Example

```java
BinaryOperator<Integer> min =
        BinaryOperator.minBy(Integer::compareTo);

System.out.println(min.apply(10, 25));
```

Output

```java
10
```

---

# Real-world Usage

`BinaryOperator` is commonly used when two values need to be combined into one.

Common production use cases include:

- Mathematical calculations
- Merging objects
- Combining Strings
- Stream reduction operations
- Finding maximum and minimum values

Example

```java
List<Integer> numbers = List.of(10, 20, 30);

Integer sum = numbers.stream()
                     .reduce(0, Integer::sum);
```

Internally, `reduce()` uses a `BinaryOperator` to combine two values into one.

---

# Best Practices

- Prefer `BinaryOperator` over `BiFunction<T, T, T>` when all types are identical.
- Keep BinaryOperators focused on a single combining operation.
- Use Method References whenever possible.
- Prefer built-in methods such as `Integer::sum` for common operations.

---

# Common Mistakes

- Using `BinaryOperator` when the output type is different.
- Forgetting that it extends `BiFunction`.
- Confusing `BinaryOperator` with `BiFunction`.

# Interview Scenario / Real-world Motivation

**Scenario: Calculating Order Total**

Suppose an e-commerce application stores multiple item prices.

Instead of manually adding them one by one, a BinaryOperator can combine them.

```java
BinaryOperator<Double> total =
        (a, b) -> a + b;
```

This same BinaryOperator can be passed to Stream API's `reduce()` method to calculate the total order amount.

Since both inputs and the output are `Double`, `BinaryOperator` is the most expressive choice.

- `BinaryOperator<T>` extends `BiFunction<T, T, T>`.
- It accepts two inputs of the same type.
- It returns the same type.

- `maxBy()` and `minBy()` are useful utility methods.
