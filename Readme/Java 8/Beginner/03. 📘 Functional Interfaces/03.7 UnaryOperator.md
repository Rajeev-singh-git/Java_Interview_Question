# 3.7 UnaryOperator

`UnaryOperator<T>` is a specialized Functional Interface that represents an operation on **one operand** where the **input and output are of the same type**.

It is a specialization of `Function<T, T>`.

Whenever an operation transforms an object **without changing its type**, `UnaryOperator` is usually the better choice.

---

# Why was UnaryOperator introduced?

Many transformations accept one input and return the same type.

Examples include:

- Convert a String to uppercase.
- Trim a String.
- Increment a number.
- Square a number.
- Normalize text.

Although these operations can be implemented using `Function<T, T>`, Java provides `UnaryOperator` because it clearly expresses the intention that the input and output are the same type.

---

# What Problem Does UnaryOperator Solve?

Suppose we want to convert a String to uppercase.

Using `Function`

```java
Function<String, String> upper =
        String::toUpperCase;
```

The input and output types are both `String`.

Instead of repeating the same type twice, Java provides `UnaryOperator`.

```java
UnaryOperator<String> upper =
        String::toUpperCase;
```

The code becomes simpler and more expressive.

---


Whenever your logic modifies an object but keeps the same type, think **UnaryOperator**.

# Definition

`UnaryOperator<T>` is a built-in Functional Interface available in the `java.util.function` package.

It accepts one input and returns the same type.

It extends `Function<T, T>`.

---

# Method Signature

```java
@FunctionalInterface
public interface UnaryOperator<T>
        extends Function<T, T> {
}
```

Since it extends `Function`, its Single Abstract Method is:

```java
T apply(T t);
```

---

# Examples

## Example 1 : Convert to Uppercase

```java
UnaryOperator<String> upper =
        String::toUpperCase;

System.out.println(upper.apply("java"));
```

Output

```
JAVA
```

---

## Example 2 : Trim a String

```java
UnaryOperator<String> trim =
        String::trim;

System.out.println(trim.apply(" Java "));
```

Output

```
Java
```

---

## Example 3 : Square a Number

```java
UnaryOperator<Integer> square =
        n -> n * n;

System.out.println(square.apply(5));
```

Output

```
25
```

---

## Example 4 : Increment a Number

```java
UnaryOperator<Integer> increment =
        n -> n + 1;

System.out.println(increment.apply(10));
```

Output

```
11
```

# Real-world Usage

`UnaryOperator` is commonly used when data needs to be modified while preserving its type.

Common production use cases include:

- Text normalization
- Number calculations
- Data sanitization
- Updating immutable objects
- Stream transformations where input and output types remain the same

Example

```java
List<String> names =
        List.of("java", "spring", "boot");

names.stream()
     .map(String::toUpperCase)
     .forEach(System.out::println);
```

Here, `map()` expects a `Function`, but since both the input and output are `String`, a `UnaryOperator<String>` can also be used.

---

# Best Practices

- Prefer `UnaryOperator` over `Function<T, T>` when both input and output types are identical.
- Keep operations focused on a single transformation.
- Prefer Method References when they improve readability.
- Avoid complex business logic inside a UnaryOperator.

---

# Common Mistakes

- Using `UnaryOperator` when the output type is different.
- Assuming it has additional methods beyond `Function`.
- Forgetting that it inherits `apply()`, `compose()`, `andThen()`, and `identity()` from `Function`.

---

# Interview Questions

### 1. What is UnaryOperator?

A specialized Functional Interface that accepts one input and returns the same type.

---

### 2. Which Functional Interface does UnaryOperator extend?

```java
Function<T, T>
```

---

### 3. When should UnaryOperator be used?

Whenever the input and output are of the same type.

---

### 4. What is the SAM of UnaryOperator?

```java
T apply(T t);
```

---

### 5. Why does Java provide UnaryOperator when Function already exists?

Because it improves readability by clearly expressing that the input and output types are identical.

---

### 6. Can UnaryOperator use compose() and andThen()?

Yes.

Since it extends `Function`, it inherits all of its default methods.

# Interview Scenario / Real-world Motivation

**Scenario: Cleaning User Input**

Suppose users submit names with extra spaces and inconsistent capitalization.

```java
UnaryOperator<String> normalize =
        name -> name.trim().toUpperCase();
```

Instead of repeating this logic throughout the application, it can be reused wherever user names need to be standardized.

Since the input is a `String` and the output is also a `String`, `UnaryOperator` clearly communicates the intent.

# Final Takeaway

> **Whenever your logic transforms a value but keeps the same type, think `UnaryOperator`. It is simply a more expressive specialization of `Function<T, T>`.**

# Key Points to Remember

- `UnaryOperator<T>` extends `Function<T, T>`.
- It accepts one input and returns the same type.
- `apply()` is its Single Abstract Method.
- It inherits `compose()`, `andThen()`, and `identity()` from `Function`.
- Prefer `UnaryOperator` over `Function<T, T>` when the input and output types are identical.
