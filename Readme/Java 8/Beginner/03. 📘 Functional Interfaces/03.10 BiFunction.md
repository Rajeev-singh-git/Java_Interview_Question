# 3.10 BiFunction

`BiFunction<T, U, R>` is a built-in Functional Interface that represents a **transformation** involving **two input values**.

It is the **two-input version of `Function`**.

It accepts two inputs and returns one output.

Whenever a transformation depends on two different values, `BiFunction` is usually the appropriate choice.

---

# Why was BiFunction introduced?

Many business operations require two input values to produce one result.

Examples include:

- Calculate total price using quantity and price.
- Concatenate first name and last name.
- Create an Employee object using name and salary.
- Calculate the distance between two locations.
- Merge two values into another object.

Although these operations can be implemented using custom methods, Java provides `BiFunction` to represent reusable transformations involving **two inputs**.

---

# What Problem Does BiFunction Solve?

Suppose we want to calculate the total price.

Without `BiFunction`

```java
public double calculateTotal(int quantity, double price) {
    return quantity * price;
}
```

Using `BiFunction`

```java
BiFunction<Integer, Double, Double> total =
        (quantity, price) -> quantity * price;
```

The transformation becomes reusable and can be passed wherever a `BiFunction` is expected.

---

# Definition

`BiFunction<T, U, R>` is a built-in Functional Interface available in the `java.util.function` package.

It is the **two-input version of `Function`**.

It accepts two inputs and returns one output.

---

# Method Signature

```java
@FunctionalInterface
public interface BiFunction<T, U, R> {

    R apply(T t, U u);

}
```

Meaning

```java
Input 1

Input 2

↓

Transform

↓

Output
```

---

# The apply() Method

The `apply()` method performs the transformation and returns the result.

Syntax

```java
R apply(T t, U u);
```

---

# Examples

## Example 1 : Calculate Total Price

```java
BiFunction<Integer, Double, Double> total =
        (quantity, price) -> quantity * price;

System.out.println(total.apply(5, 100.0));
```

Output

```
500.0
```

---

## Example 2 : Join First and Last Name

```java
BiFunction<String, String, String> fullName =
        (first, last) -> first + " " + last;

System.out.println(fullName.apply("John", "Doe"));
```

Output

```
John Doe
```

---

## Example 3 : Create Employee Object

```java
BiFunction<String, Double, Employee> createEmployee =
        Employee::new;
```

---

## Example 4 : Calculate Maximum

```java
BiFunction<Integer, Integer, Integer> maximum =
        Math::max;

System.out.println(maximum.apply(10, 20));
```

Output

```
20
```

---

# Built-in Methods

Like `Function`, `BiFunction` provides one useful default method.

---

## andThen()

Applies the current `BiFunction` first and then applies a `Function` to the result.

```java
BiFunction<Integer, Integer, Integer> add =
        Integer::sum;

Function<Integer, String> convert =
        Object::toString;

BiFunction<Integer, Integer, String> result =
        add.andThen(convert);

System.out.println(result.apply(10, 20));
```

Output

```
30
```

---

# Real-world Usage

`BiFunction` is commonly used whenever two values need to be transformed into one result.

Common production use cases include:

- Price calculation
- DTO creation
- Object construction
- String concatenation
- Mathematical calculations
- Data transformation

---

# Best Practices

- Keep each BiFunction focused on a single transformation.
- Prefer Method References whenever they improve readability.
- Use meaningful names such as `calculateTotal`, `createEmployee`, or `buildFullName`.
- Use `andThen()` when additional processing is required.

---

# Common Mistakes

- Using `BiFunction` when only one input is required.
- Writing very complex transformations inside `apply()`.
- Forgetting that `andThen()` returns another `BiFunction`.
- Using `BiFunction` when a boolean result is actually required.

# Interview Scenario / Real-world Motivation

### Scenario: Employee Creation

Suppose an HR system receives an employee's name and salary.

Instead of repeatedly writing object creation logic, we can define a reusable `BiFunction`.

```java
BiFunction<String, Double, Employee> createEmployee =
        Employee::new;
```

Whenever a new employee needs to be created,

```java
Employee employee =
        createEmployee.apply("Alice", 75000.0);
```

The same transformation can now be reused throughout the application.

# Final Takeaway

> **Whenever your logic transforms two input values into one result, think `BiFunction`. It is the two-input version of `Function`, allowing reusable transformations involving two inputs.**

---

# Key Points to Remember

- `BiFunction<T, U, R>` is the two-input version of `Function`.
- It accepts two inputs.
- It returns one output.
- `apply()` is its Single Abstract Method.
- `andThen()` performs additional processing on the result.
- It is commonly used for object creation, DTO mapping, price calculation, and other two-input transformations.
