# 3.5 Consumer

`Consumer<T>` is one of Java's built-in Functional Interfaces.

It represents an **operation** that accepts one input and **does not return any result**.

Unlike `Function`, which transforms data, `Consumer` simply **performs an action** on the supplied input.

It is commonly used for **printing**, **logging**, **saving data**, **sending notifications**, and many other side-effect operations.

---

# Why was Consumer introduced?

Many applications perform actions without returning a value.

Examples include:

- Print employee details.
- Save an employee to the database.
- Send an email.
- Log a message.
- Store data in a file.

Before Java 8, these operations were usually written as separate methods or anonymous inner classes.

Java 8 introduced `Consumer` so that actions could be treated as reusable behavior and passed wherever needed.

---

# What Problem Does Consumer Solve?

Suppose we have an `Employee`.

Sometimes we don't need to transform it or check a condition.

We simply want to perform an action.

Examples:

- Print Employee
- Save Employee
- Send Email
- Log Employee

Instead of creating separate methods everywhere, Java allows us to pass the action itself.

```java
employee -> System.out.println(employee)
```

The API simply executes whatever action you provide.

| Predicate          | Function         | Consumer           |
| ------------------ | ---------------- | ------------------ |

# The accept() Method

The `accept()` method performs the required action.

Syntax

```java
void accept(T t);
```

Example

```java

        System.out::println;

printer.accept("Hello");
```

Output

```

```

# Built-in Methods

Besides `accept()`, `Consumer` provides one useful default method.

---

## andThen()

Executes another Consumer after the current Consumer.

```java
'Input

↓

Consumer A

↓

Consumer B
```

Example

```java
Consumer<String> print =
        System.out::println;

Consumer<String> upper =
        str -> System.out.println(str.toUpperCase());

Consumer<String> result =
        print.andThen(upper);

result.accept("java");
```

Output

```java
java
JAVA
```

`andThen()` is useful when multiple actions need to be performed on the same input.

---

# Real-world Usage

`Consumer` is widely used whenever an application needs to **perform an action** without returning a value.

Common production use cases include:

- Stream API (`forEach()`)
- Logging
- Sending emails or notifications
- Writing to files
- Saving data
- Updating records
- Event handling
- Spring Framework APIs

---

## Stream API

The `forEach()` method expects a `Consumer`.

```java
List<String> names = List.of("Alice", "Bob", "Charlie");

names.stream()
     .forEach(System.out::println);
```

Output

```
Alice
Bob
Charlie
```

Here,

- `forEach()` expects a `Consumer`
- The Consumer performs an action for every element
- Nothing is returned

---

## Logging

```java
Consumer<String> logger =
        msg -> System.out.println("[INFO] " + msg);

logger.accept("Application Started");
```

Output

```
[INFO] Application Started
```

---

## Saving Data

```java
Consumer<Employee> saveEmployee =
        employeeRepository::save;
```

The Consumer performs the save operation.

No value is returned.

---

## Sending Notifications

```java
Consumer<String> sendEmail =
        emailService::send;
```

The Consumer performs the action of sending an email.

# Best Practices

- Keep Consumers focused on a single action.
- Prefer Method References when they improve readability.
- Avoid placing complex business logic inside Consumers.
- Use `andThen()` to chain multiple actions.
- Give Consumers meaningful names such as `printer`, `logger`, or `saveEmployee`.

---

# Common Mistakes

- Using `Consumer` when a value should be returned (use `Function` instead).
- Expecting `accept()` to return a result.
- Writing multiple unrelated actions inside a single Consumer.
- Forgetting that `andThen()` executes Consumers sequentially.

---

# Common Interview Traps

- `Consumer` always returns `void`.
- `accept()` is the Single Abstract Method (SAM).
- `forEach()` expects a `Consumer`.
- `andThen()` executes Consumers in sequence.
- Consumers are designed for **side effects**, not transformations.

# Interview Questions

### 1. What is Consumer?

`Consumer<T>` is a Functional Interface that accepts one input and performs an action without returning a value.

---

### 2. When should Consumer be used?

Whenever an operation performs an action but does not produce a result.

---

### 3. What is the SAM of Consumer?

```java
void accept(T t);
```

---

### 4. What is the purpose of `accept()`?

It performs the required action on the supplied input.

---

### 5. What is the difference between Function and Consumer?

| Function                 | Consumer         |
| ------------------------ | ---------------- |
| Returns a value          | Returns nothing  |
| Used for transformations | Used for actions |

---

### 6. What does `andThen()` do?

It executes the current Consumer first and then executes another Consumer on the same input.

---

### 7. Why does Consumer return `void`?

Because its purpose is to perform an action rather than produce a value.

---

### 8. Where is Consumer commonly used?

- Stream API (`forEach()`)
- Logging
- Notifications
- Database operations
- Event handling
- Spring Framework

---

### 9. Can Consumers be chained?

Yes.

Using:

```java
consumer1.andThen(consumer2);
```

---

### 10. Why is Consumer useful?

It allows actions to become reusable, composable, and easy to pass to APIs.

# Interview Scenario / Real-world Motivation

**Scenario: Processing Orders**

Suppose an e-commerce application retrieves a list of orders.

Each order needs to:

- Print the order details
- Save an audit log
- Send a notification

Instead of writing one large method, we can define separate Consumers.

```java
Consumer<Order> printOrder = System.out::println;

Consumer<Order> audit =
        order -> auditService.save(order);

Consumer<Order> notifyCustomer =
        order -> emailService.send(order);

Consumer<Order> processOrder =
        printOrder
            .andThen(audit)
            .andThen(notifyCustomer);
```

Each Consumer performs a single responsibility.

`andThen()` combines them into a reusable processing pipeline.

This pattern is commonly used in enterprise applications.

# What's Next?

A `Consumer` performs an action but does not return a value.

But what if we need to **generate a value without taking any input**?

For example:

# Final Takeaway

> **Whenever your logic performs an action without returning a value, think `Consumer`. It represents a reusable operation that accepts one input and produces side effects instead of a result.**

---

# Key Points to Remember

- `Consumer<T>` accepts one input.
- It always returns `void`.
- `accept()` is its Single Abstract Method.
- `andThen()` chains multiple Consumers.
- `Consumer` is heavily used in Stream API's `forEach()` method.
- It is commonly used for logging, notifications, persistence, and event handling.
- Use `Consumer` for actions, not transformations.
