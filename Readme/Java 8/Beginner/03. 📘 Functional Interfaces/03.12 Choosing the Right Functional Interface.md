# 3.12 Choosing the Right Functional Interface

After learning all the Functional Interfaces, the most important interview skill is knowing **which one to use**.

Use the following decision guide whenever you need to choose a Functional Interface.

---

# Decision Tree

```
Need to...

        │
        ▼
Check a condition?
        │
       Yes
        ▼
Predicate

--------------------

Transform data?
        │
       Yes
        ▼
Function

--------------------

Perform an action?
        │
       Yes
        ▼
Consumer

--------------------

Generate a value?
        │
       Yes
        ▼
Supplier
```

---

# Core Functional Interfaces

| Requirement | Functional Interface | SAM |
|-------------|----------------------|-----|
| Check a condition | `Predicate<T>` | `boolean test(T t)` |
| Transform data | `Function<T, R>` | `R apply(T t)` |
| Perform an action | `Consumer<T>` | `void accept(T t)` |
| Generate a value | `Supplier<T>` | `T get()` |

---

# Specialized Functional Interfaces

| Requirement | Functional Interface | SAM |
|-------------|----------------------|-----|
| Transform one object into the same type | `UnaryOperator<T>` | `T apply(T t)` |
| Combine two objects of the same type | `BinaryOperator<T>` | `T apply(T t1, T t2)` |
| Check a condition using two inputs | `BiPredicate<T, U>` | `boolean test(T t, T u)` |
| Transform two inputs into one result | `BiFunction<T, U, R>` | `R apply(T t, U u)` |
| Perform an action using two inputs | `BiConsumer<T, U>` | `void accept(T t, U u)` |

---

# Common JDK Usage

| API | Functional Interface |
|------|----------------------|
| `Stream.filter()` | `Predicate` |
| `Stream.map()` | `Function` |
| `Stream.forEach()` | `Consumer` |
| `Stream.generate()` | `Supplier` |
| `Stream.reduce()` | `BinaryOperator` |
| `Map.forEach()` | `BiConsumer` |

---

# Interview Cheat Sheet

| If you need to... | Use |
|-------------------|-----|
| Filter data | `Predicate` |
| Convert one object into another | `Function` |
| Print, save, or log | `Consumer` |
| Create or generate an object | `Supplier` |
| Modify an object but keep the same type | `UnaryOperator` |
| Combine two objects of the same type | `BinaryOperator` |
| Validate two values | `BiPredicate` |
| Transform two values | `BiFunction` |
| Process two values | `BiConsumer` |

---

# Key Points to Remember

- Functional Interfaces enable Lambda Expressions and Method References.
- A Functional Interface contains exactly one abstract method (SAM).
- Choose a Functional Interface based on **what your code needs to do**, not by memorizing method names.
- Prefer Java's built-in Functional Interfaces over creating custom ones.
- Use specialized Functional Interfaces (`UnaryOperator`, `BinaryOperator`, etc.) when they better express your intent.

---

# Final Takeaway

> **Don't memorize Functional Interfaces individually. Instead, identify whether your problem is about checking, transforming, performing, generating, or combining, and the correct Functional Interface becomes obvious.**
