# 5. Default & Static Methods

Java 8 introduced **Default Methods** and **Static Methods** in interfaces.

Before Java 8, interfaces could only contain abstract methods. This made it difficult to evolve existing APIs because adding a new abstract method would break every implementing class.

Default Methods and Static Methods solved this problem while maintaining **backward compatibility**.

- **Default Methods** → *API evolution without breaking existing implementations.*
- **Static Methods** → *Keeping interface-related utility and factory methods inside the interface (e.g., `List.of()`, `Map.of()`) instead of separate utility classes.*

---

# Why were Default Methods Introduced?

Suppose an interface is widely used.

```java
interface Vehicle {

    void start();

}
```

Now suppose we want to add a new method.

```java
void stop();
```

Before Java 8, every class implementing `Vehicle` would be forced to implement `stop()`.

This would break existing applications.

Java 8 introduced **Default Methods** so interfaces could evolve by adding new functionality **without breaking existing implementations**.

---

## Default Methods

A **Default Method** is a method inside an interface that provides a default implementation.

It is declared using the `default` keyword.

```java
interface Vehicle {

    void start();

    default void stop() {
        System.out.println("Vehicle stopped");
    }

}
```

Implementing classes automatically inherit the default implementation.

```java
class Car implements Vehicle {

    @Override
    public void start() {
        System.out.println("Car started");
    }

}
```

Usage

```java
Car car = new Car();

car.stop();
```

Output

```java
Vehicle stopped
```

---

### Overriding Default Methods

Implementing classes can override a Default Method whenever a custom implementation is required.

```java
class Car implements Vehicle {

    @Override
    public void start() {
        System.out.println("Car started");
    }

    @Override
    public void stop() {
        System.out.println("Car stopped");
    }

}
```

---

## Static Methods

Interfaces can also contain static methods.

Unlike Default Methods, Static Methods belong to the **interface itself**, not to its implementing classes.

```java
interface Vehicle {

    static void info() {
        System.out.println("Vehicle Interface");
    }

}
```

They must always be called using the interface name.

```java
Vehicle.info();
```

Static Methods are **not inherited** by implementing classes.

```java
class Car implements Vehicle {

}

Car.info();      // ❌ Compile-time error
Vehicle.info();  // ✔ Correct
```

They also **cannot be overridden**.

If an implementing class declares a static method with the same signature, it simply creates another unrelated static method instead of overriding the interface's method.

---

### Real-world Scenario: Why Static Methods in Interfaces?

Suppose you're designing a payment framework.

```java
interface PaymentProcessor {

    void processPayment(double amount);

    static boolean isValidAmount(double amount) {
        return amount > 0;
    }

}
```

Different payment processors implement the interface.

```java
class CreditCardProcessor implements PaymentProcessor {

    @Override
    public void processPayment(double amount) {
        System.out.println("Processing Credit Card Payment");
    }

}

class UpiProcessor implements PaymentProcessor {

    @Override
    public void processPayment(double amount) {
        System.out.println("Processing UPI Payment");
    }

}
```

Before processing a payment, any client can validate the amount.

```java
if (PaymentProcessor.isValidAmount(amount)) {
    processor.processPayment(amount);
}
```

Here, `isValidAmount()`:

- Is closely related to the `PaymentProcessor` interface.
- Does not depend on any object's state.
- Can be shared by all implementations.
- Does not need to be inherited or overridden.

Keeping it inside the interface makes the API more cohesive and avoids creating a separate utility class.

> **Note:** The JDK follows the same approach. Methods such as `List.of()`, `Set.of()`, and `Map.of()` are static methods defined inside their respective interfaces.

---

# Default Method vs Static Method

| Default Method                    | Static Method                              |
| --------------------------------- | ------------------------------------------ |
| Uses the `default` keyword        | Uses the `static` keyword                  |
| Inherited by implementing classes | Not inherited                              |
| Can be overridden                 | Cannot be overridden                       |
| Called using an object            | Called using the interface name            |
| Used to evolve interfaces         | Used for interface-related utility methods |

---

# Multiple Default Methods (Diamond Problem)

A class may implement multiple interfaces that contain the same Default Method.

```java
interface A {

    default void show() {
        System.out.println("A");
    }

}

interface B {

    default void show() {
        System.out.println("B");
    }

}
```

This creates ambiguity.

```java
class Demo implements A, B {

    @Override
    public void show() {
        System.out.println("Resolved");
    }

}
```

The implementing class **must override** the conflicting Default Method.

---

## Calling a Specific Default Method

Inside the overriding method, a specific interface implementation can be invoked using the `InterfaceName.super` syntax.

```java
class Demo implements A, B {

    @Override
    public void show() {
        A.super.show();
    }

}
```

Similarly,

```java
B.super.show();
```

invokes the implementation from interface `B`.

---

# Real-world Usage

### Default Methods

Default Methods are primarily used to evolve existing APIs without breaking implementations.

Examples include:

- Java Collections Framework
- Stream API enhancements
- Third-party libraries
- Framework interfaces

---

### Static Methods

Static Methods are used for utility operations that are closely related to an interface but do not require object state.

Examples include:

- `List.of()`
- `Set.of()`
- `Map.of()`
- Validation methods
- Factory methods
- Helper methods related to the interface

---

# Best Practices

- Use Default Methods only when a meaningful default implementation exists.
- Keep Default Methods simple and reusable.
- Use Static Methods for interface-specific utility operations.
- Keep Static Methods stateless and independent of object instances.
- Avoid placing complex business logic inside interfaces.

---

# Common Mistakes

- Assuming Static Methods are inherited by implementing classes.
- Assuming Static Methods can be overridden.
- Forgetting to resolve conflicting Default Methods.
- Using Default Methods when every implementation requires different behavior.

# Interview Questions

### 1. Why were Default Methods introduced?

To allow interfaces to evolve by adding new functionality without breaking existing implementations.

---

### 2. Why were Static Methods introduced in interfaces?

To keep utility methods closely related to an interface instead of creating separate utility classes.

---

### 3. What is the difference between a Default Method and a Static Method?

Default Methods are inherited and can be overridden.

Static Methods belong to the interface, are not inherited, and cannot be overridden.

---

### 4. What happens if two interfaces define the same Default Method?

The implementing class must override the method to resolve the ambiguity.

---

### 5. Can a Static Method in an interface be overridden?

No.

If a class declares a static method with the same signature, it creates a new unrelated static method instead of overriding the interface's method.

---

### 6. How do you invoke a specific Default Method when multiple interfaces define it?

Using the `InterfaceName.super` syntax.

```java
A.super.show();
```

---

### 7. Give a real-world example of Static Methods in interfaces.

Methods such as `List.of()`, `Set.of()`, and `Map.of()` are static methods defined inside JDK interfaces.
