# 2. Lambda (λ) Expression

Lambda Expression is one of the most significant features introduced in **Java 8**. It brings **Functional Programming** to Java by allowing **behavior** to be passed as an argument through **Functional Interfaces**.

Lambda Expressions replace verbose anonymous inner classes with a concise syntax, making code cleaner, easier to read, and easier to maintain.

They are heavily used throughout modern Java, especially with the **Stream API**, **Collections Framework**, **CompletableFuture**, and many popular frameworks.

---

# Why were Lambda Expressions introduced?

Before Java 8, implementing a Functional Interface required creating an anonymous inner class.

```java
Runnable r = new Runnable() {
    @Override
    public void run() {
        System.out.println("Hello");
    }
};
```

Although the implementation contains only one method, a considerable amount of boilerplate code is required.

Java 8 introduced Lambda Expressions to simplify this.

```java
Runnable r = () -> System.out.println("Hello");
```

Benefits:

- Less boilerplate
- Better readability
- More expressive code
- Encourages Functional Programming

---

# What is a Lambda Expression?

A Lambda Expression is an **anonymous function**.

It is called anonymous because it does **not have**:

- Method Name
- Return Type
- Access Modifier

A Lambda Expression contains only:

- Parameters (if any)
- Arrow Operator (`->`)
- Method Body

---

# Mental Model

Imagine a method that has no name.

Instead of creating a class just to implement one method, you directly pass the behavior itself.

> **Lambda = Passing behavior as data.**

Traditional Java

```
Behavior
    │
    ▼
Object
```

Java 8

```
Behavior
    │
    ▼
Passed directly
```

This ability to treat behavior like data is what enables Functional Programming in Java.

---

# General Syntax

```java
(parameter_list) -> expression
```

or

```java
(parameter_list) -> {
    // multiple statements
}
```

---

# Examples

## Example 1: No Parameters

Without Lambda

```java
public void greet() {
    System.out.println("Hello");
}
```

Lambda

```java
() -> System.out.println("Hello");
```

---

## Example 2: Multiple Parameters

Without Lambda

```java
public void add(int a, int b) {
    System.out.println(a + b);
}
```

Lambda

```java
(a, b) -> System.out.println(a + b);
```

The compiler can infer parameter types, so writing

```java
(int a, int b)
```

is usually unnecessary.

---

## Example 3: Returning a Value

Without Lambda

```java
public String getName(String name) {
    return name;
}
```

Lambda

```java
name -> name
```

Since there is only one parameter:

- Parentheses become optional.
- Parameter type can usually be omitted.

---

# Characteristics

### 1. Lambda Expressions can have zero or more parameters.

```java
() -> System.out.println("Hello");

a -> System.out.println(a);

(a, b) -> System.out.println(a + b);
```

---

### 2. Parameter types are optional if the compiler can infer them.

```java
(int a, int b) -> a + b;

(a, b) -> a + b;
```

---

### 3. Parentheses are optional for a single parameter.

```java
a -> a * a
```

---

### 4. Empty parentheses are mandatory when there are no parameters.

```java
() -> System.out.println("Hello");
```

---

### 5. Curly braces are optional for a single statement.

```java
x -> x * x
```

```java
x -> {
    return x * x;
}
```

---

### 6. If the body contains only a return statement, both braces and the `return` keyword can be omitted.

```java
x -> x * x
```

instead of

```java
x -> {
    return x * x;
}
```

---

### 7. A Lambda Expression is not executed by itself.

It must be assigned to or passed where a **Functional Interface** is expected.

```java
Runnable r = () -> System.out.println("Hello");
```

Here:

- `Runnable` is a Functional Interface.
- The Lambda provides the implementation of its `run()` method.

---

# How Lambda Works

A Lambda Expression does **not** create a new method.

Instead, the compiler maps it to the **Single Abstract Method (SAM)** of a Functional Interface.

```
Lambda Expression
        │
        ▼
Compiler
        │
        ▼
Functional Interface (SAM)
        │
        ▼
JVM (invokedynamic)
        │
        ▼
Executable implementation
```

Modern JVMs use the `invokedynamic` instruction to efficiently implement Lambda Expressions.

> **Note:** We'll learn Functional Interfaces in detail in the next chapter.

---

# Variable Capture

A Lambda Expression can access local variables from its enclosing scope only if they are **final** or **effectively final**.

```java
int x = 10;

Runnable r = () -> System.out.println(x);

// x = 20;   // Compilation Error
```

This restriction ensures predictable behavior when Lambdas capture local variables.

> **Note:** We'll understand the reason behind this restriction in a later chapter.

---

# Lambda vs Anonymous Inner Class

| Anonymous Inner Class | Lambda Expression |
|------------------------|-------------------|
| Verbose | Concise |
| Requires more boilerplate | Less boilerplate |
| Creates an anonymous class | Implements only the Functional Interface's SAM |
| Has its own `this` reference | Uses the enclosing class's `this` |
| Suitable for multiple methods | Suitable only for Functional Interfaces |
| Harder to read | Easier to read |

---

# Advantages

- Reduces boilerplate code.
- Improves readability.
- Makes code concise.
- Enables Functional Programming.
- Encourages passing behavior instead of creating classes.
- Works seamlessly with the Stream API and modern Java APIs.

---

# Limitations

- Can only be used with Functional Interfaces.
- Cannot implement interfaces having multiple abstract methods.
- Not suitable for very complex business logic.
- Overusing Lambdas can reduce readability.

---

# Real-world Usage

One of the biggest advantages of Lambda Expressions is that developers can focus on **what** needs to be done rather than **how** it is implemented.

For example, sorting a collection changed from verbose anonymous inner classes to concise behavior passed directly to an API.

Before Java 8

```java
Collections.sort(list, new Comparator<Employee>() {
    @Override
    public int compare(Employee e1, Employee e2) {
        return e1.getSalary() - e2.getSalary();
    }
});
```

With Lambda

```java
list.sort((e1, e2) -> e1.getSalary() - e2.getSalary());
```

Common production use cases include:

- Stream API pipelines
- Sorting using `Comparator`
- `ExecutorService`
- `Runnable`
- `CompletableFuture`
- Event listeners
- Spring Framework APIs

---

# Best Practices

- Keep Lambda Expressions short and readable.
- Prefer Method References when they improve readability.
- Avoid writing complex business logic inside Lambdas.
- Avoid unnecessary side effects.
- Use meaningful parameter names.

---

# Common Mistakes

- Using Lambda Expressions with non-functional interfaces.
- Writing very long Lambdas that reduce readability.
- Mixing complex business logic with Lambda expressions.
- Ignoring readability in favor of writing fewer lines of code.

---

# Common Interview Traps

- Lambda Expressions work only with Functional Interfaces.
- A Lambda does not have its own `this`; it uses the enclosing class instance.
- Local variables captured inside a Lambda must be final or effectively final.
- Prefer Method References when they improve readability.

# Interview Questions

### 1. Why were Lambda Expressions introduced?

Lambda Expressions were introduced to reduce boilerplate code, improve readability, and bring Functional Programming capabilities to Java.

---

### 2. What is a Lambda Expression?

A Lambda Expression is an anonymous function that provides the implementation of the Single Abstract Method (SAM) of a Functional Interface.

---

### 3. Can a Lambda Expression exist without a Functional Interface?

No.

A Lambda Expression always targets a **Functional Interface**.

---

### 4. Can Lambda Expressions have multiple parameters?

Yes.

```java
(a, b) -> a + b
```

---

### 5. Can a Lambda Expression contain multiple statements?

Yes.

```java
(a, b) -> {
    int sum = a + b;
    return sum;
}
```

---

### 6. Why can Lambda Expressions be used only with Functional Interfaces?

Because the compiler needs exactly one abstract method to map the Lambda Expression to.

If an interface contains multiple abstract methods, the compiler cannot determine which method the Lambda should implement.

---

### 7. Why must captured local variables be final or effectively final?

Because Lambda Expressions capture local variables from the enclosing scope.

Java requires these variables to remain unchanged to ensure consistent and predictable behavior.

---

### 8. Does a Lambda Expression have its own `this` reference?

No.

Inside a Lambda Expression, `this` refers to the enclosing class instance.

---

### 9. Does a Lambda Expression create an anonymous inner class?

Not necessarily.

Modern JVMs typically use the `invokedynamic` instruction to create an efficient implementation instead of generating an anonymous inner class.

---

### 10. What is the biggest advantage of Lambda Expressions?

They allow developers to pass behavior directly, replacing verbose anonymous inner classes with concise, readable code.

---

### 11. What is the difference between a Lambda Expression and an Anonymous Inner Class?

A Lambda Expression provides the implementation of a Functional Interface's Single Abstract Method (SAM), whereas an Anonymous Inner Class creates a complete anonymous class.

---

### 12. When should you avoid using Lambda Expressions?

Avoid Lambdas when:

- The logic becomes long or difficult to understand.
- Readability suffers.
- A normal method provides a cleaner solution.
